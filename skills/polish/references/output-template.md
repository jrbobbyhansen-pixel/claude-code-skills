# Output Contract ‚Äî desk findings JSON, POLISH.md, inline summary

There are two artifacts: (1) what each **desk agent writes** (a JSON file), and (2) what **`aggregate.py` renders**
(`POLISH.md` + sidecar). Desks never write `POLISH.md` directly ‚Äî the script assigns IDs, dedupes, verifies, scores,
and renders deterministically.

## 1. What each desk agent writes  ‚Üí `.polish/findings/<desk>-<slice>.json`
```json
{
  "desk": "MOTION",
  "slice": "src-components",
  "covered_files": ["src/components/Button.tsx", "src/components/Card.tsx"],
  "receipts": {                                 // REQUIRED: one per covered file ‚Äî its first non-empty
    "src/components/Button.tsx": "import React from 'react';",   // line, verbatim (the read receipt);
    "src/components/Card.tsx":   "import { View } from 'react-native';"  // no/bad receipt ‚Üí file counts UNSWEPT
  },
  "findings": [
    {
      "file":  "src/components/Button.tsx",
      "line":  42,
      "anchor": "activeOpacity={0.2}",          // REQUIRED: 5‚Äì12 word verbatim snippet AT that line (read-proof)
      "what":  "Primary CTA flashes at activeOpacity 0.2 instead of a scale press",
      "why":   "A subtle scale reads as tactile; a hard opacity flash reads as 'web in a wrapper'.",
      "who":   "Apple ‚Äî controls respond on touch-down with a slight scale [PRINCIPLE]",
      "fix":   "Swap TouchableOpacity‚ÜíPressable; Reanimated scale 1‚Üí0.96 on pressIn (Reanimated is in package.json).",
      "class": "CONVENTION",                      // OBJECTIVE | CONVENTION | TASTE  (see doctrine)
      "change_type": "press-feedback",            // dedup key (file,line,change_type)
      "fix_target": "activeOpacity",              // optional ‚Äî enables conflict detection on a shared property
      "flags": [],                                // [] | ["NEW CODE"] | ["REQUIRES DEP: react-native-reanimated"]
      "confidence": 0.8,                          // 0.5‚Äì1.0; <0.6 ‚Üí collapsed Minor group
      "effort": "low"                             // low | med | high  (NEW CODE/REQUIRES DEP auto-bumped)
    }
  ],
  "out_of_scope": [
    { "file": "src/navigation/RootStack.tsx", "what": "reorder tabs / add onboarding route", "axis": "1 (nav/IA)" }
  ]
}
```
**Hard rules for the agent:**
- `file`,`line`,`anchor`,`what`,`why`,`who`,`fix`,`class` are required ‚Äî a finding missing any goes to `Malformed`.
- `anchor` must be quotable from the real line, or `aggregate.py` rejects the finding as a phantom.
- `who` carries a `[DOCUMENTED]`/`[PRINCIPLE]` tag, or `‚Äî (principle, no exemplar)`. Never an invented number.
- `class` per doctrine: OBJECTIVE (measurable gap) / CONVENTION (breaks app's own system) / TASTE (preference; offer-worded).
- `covered_files` MUST list every file the agent actually read in its slice, and `receipts` MUST carry each file's
  first non-empty line verbatim ‚Äî `aggregate.py` checks receipts against disk; a claimed file with no matching
  receipt is counted UNSWEPT. Coverage is proven, never claimed.
- `[DOCUMENTED]` numbers are machine-checked against `north-stars.md`; an unblessed number auto-downgrades the WHO
  to `[PRINCIPLE]` and flags the finding `CITATION-DOWNGRADED`.
- Do NOT set `id` or `tier` ‚Äî the script computes both. Do NOT write `POLISH.md`.

### Checkbox / ID semantics
- IDs are content-hash stable (`<PREFIX>-<sha1(desk|relpath|what)[:6]>`) ‚Äî they survive line shifts, so the apply
  menu (`pick MO-7f3a91 ‚Ä¶`), per-finding revert, and the stateful delta all key on the ID, never the line number.
- `- [ ]` open ¬∑ `- [x]` applied + passed gate ¬∑ `- [!]` applied then reverted (gate failed) ‚Äî with `‚Ü≥ REVERTED: <err>`.
- Findings the user **declines** are recorded in `state.json` `declined{id: reason}` ‚Äî the graveyard. They render
  collapsed under "Previously declined" and are never re-proposed on later runs.
